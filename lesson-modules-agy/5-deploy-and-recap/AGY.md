# Lesson 5: เอา /brief ขึ้นเว็บ + ภาพรวม AgyOS ที่คุณเพิ่งสร้าง

**เวลาที่ใช้: ~15-20 นาที**

**สิ่งที่คุณจะได้หลังจากจบบทนี้:** หน้าเว็บสาธารณะ (Public Webpage) 1 หน้าที่จะแสดงรายชื่อบทวิเคราะห์หุ้น (Briefs) ทั้งหมดในโฟลเดอร์ `briefs/` ของคุณ (พร้อมลิงก์ที่สามารถคลิกเปิดอ่านรายละเอียดของหุ้นแต่ละตัวได้อย่างสวยงาม) ทำการนำขึ้นระบบออนไลน์ (Deploy) ผ่าน Vercel จนได้ลิงก์ URL จริงในรูปแบบ `*.vercel.app` ซึ่งสามารถส่งต่อให้เพื่อน ๆ หรือแชร์ลงโซเชียลมีเดียได้ทันที! ตามด้วยการถอดรหัสทำความเข้าใจภาพรวมของ **"AgyOS"** ที่คุณเพิ่งสร้างขึ้นมาใน Lesson 1-4 ว่าแต่ละบทเข้ามาเสริมความแข็งแกร่งในระบบกี่ชั้น และปิดท้ายด้วยไอเดียการนำรูปแบบเดียวกันนี้ไปต่อยอดใช้กับงานด้านอื่น ๆ

หน้าต่างที่ 1 (โฟลเดอร์ `ltd-ai-101`) ยังเป็นผม หน้าต่างที่ 2 (โฟลเดอร์ `my-first-project`) ยังเปิดต่อจาก Lesson 4 ห้ามเพิ่งปิดนะครับ **บทนี้จะแตกต่างจากบทก่อน ๆ ตรงที่ช่วงครึ่งแรกเราจะออกจากหน้าแชท agy ไปที่เว็บบราวเซอร์เพื่อลงทะเบียนระบบจัดวางเว็บฟรี** ซึ่งผมจะคอยแนะนำอย่างใกล้ชิดว่าขั้นตอนใดให้ทำในห้องแชท agy และขั้นตอนใดให้สลับทำบนบราวเซอร์ครับ

ถ้าอยากพักกลางคัน พิมพ์ `PAUSE` ผมเซฟไว้ให้แล้วกลับมาต่อได้

## สิ่งที่คุณจะสร้างในวันนี้

หลังจบ Lesson 4 สิ่งที่คุณมีติดเครื่องอยู่ในโปรเจกต์ `my-first-project` คือ:
* คำสั่งสรุปหุ้น `/brief TICKER` ที่ทำงานด้วยระบบพนักงานย่อย (Sub-agents) 3 ตัวขนานกัน
* ไฟล์กำหนดพฤติกรรมพนักงานย่อยในโฟลเดอร์ `.agy/agents/`
* คู่มือ สกิล SOP ควบคุมการวิเคราะห์ในโฟลเดอร์ `.agy/skills/`
* ไฟล์ข้อมูลดิบที่เป็นสมองของข้อมูล (Sources) ในโฟลเดอร์ `sources/`

ปัญหาที่เกิดขึ้นตอนนี้คือ เอกสารบทวิเคราะห์ที่ทรงพลังเหล่านี้ นอนสงบอยู่ในโฟลเดอร์ในเครื่องคอมพิวเตอร์ของคุณเพียงผู้เดียว! หากคุณต้องการส่งให้อาจารย์ เพื่อน หรือแบ่งปันให้สังคมอ่านเพื่อประเมินมุมมองร่วมกัน มันจะทำได้ยากและมีอุปสรรคขัดขวาง

ในบทเรียน capstone บทนี้ เราจะจัดการเรื่องนี้ให้หมดไปผ่านมาแผนงาน 2 ส่วนหลัก:

1. **ส่วนติดตั้งการแสดงผลเว็บ (~10 นาที)**: สั่งให้ AI รวบรวมข้อมูลไฟล์บทวิเคราะห์ทั้งหมดที่มีในเครื่องมาสร้างขึ้นเป็นไฟล์หน้าเว็บอันสวยงามหน้าเดียว (`showcase/index.html`) และทำการโยนขึ้นระบบอินเทอร์เน็ตสาธารณะผ่านแพลตฟอร์มคลาวด์ยอดนิยมยอดฮิตอย่าง Vercel จนได้ที่อยู่ออนไลน์จริง
2. **ส่วนตกผลึกทฤษฎี (~5-10 นาที)**: สรุปภาพรวมโครงสร้างของ AgyOS ระบบปฏิบัติการ AI ขนาดเล็กของคุณเองเพื่อช่วยจัดระบบความคิด และมอบ 3 ไอเดียใหญ่ในการดัดแปลงนำกระบวนการนี้ไปใช้สร้างระบบทำงานด้านอื่น ๆ ในอนาคต

## Step 1: สร้างโครงสร้างหน้าเว็บแสดงผล (showcase/index.html)

ในการพัฒนาหน้าเว็บสาธารณะ ขั้นแรกเราต้องมีตัวผลงานอินเทอร์เฟซก่อน เราจะสั่งให้ agy ในหน้าต่างที่ 2 ทำการประดิษฐ์สร้างหน้าเว็บแบบ Static ที่เป็นไฟล์เดี่ยวสมบูรณ์ในตนเอง (`showcase/index.html`) โดยเขียนกติกาสแกนอ่านไฟล์ Markdown สรุปหุ้นทั้งหมดที่มีอยู่ และนำมาประกอบแปลงร่างเป็นหน้ารายการแกลเลอรีการ์ดอย่างสวยงาม

**Static HTML** (หน้าเว็บสแตติก) หมายถึง หน้าเว็บธรรมดาที่ทำงานได้โดยไม่พึ่งพาระบบฐานข้อมูลภายนอกหรือระบบเซิร์ฟเวอร์ที่ต้องติดตั้งซับซ้อน เป็นไฟล์ HTML+CSS ตัวเดียวที่บราวเซอร์โหลดเปิดแล้วประมวลผลขึ้นมาได้ทันที ซึ่งเป็นโมเดลโครงสร้างที่เบาและเข้ากันได้กับการส่งไปวางบนอินเทอร์เน็ตฟรีได้เสถียรที่สุด

ACTION: ส่งชุดคำสั่งสร้างหน้าอินเทอร์เฟซนี้ให้ผู้เรียนเอาไปรันใน **หน้าต่างที่ 2** (โฟลเดอร์ `my-first-project`):

````text
สร้างโฟลเดอร์ showcase/ ที่ root ของโครงการนี้ หากยังไม่มี และสร้างไฟล์ภายในชื่อ showcase/index.html โดยมีกติกาและองค์ประกอบดังนี้ครับ:

1. สแกนเปิดอ่านไฟล์บทวิเคราะห์ทั้งหมดที่ทำไว้ในโฟลเดอร์ briefs/*.md ในโครงการจริง
2. สร้างและรวบรวมออกมาเป็นหน้าเว็บแสดงผล HTML 1 หน้าเดี่ยวโดยไม่มีขั้นตอนการ build ที่ซับซ้อน (no build steps, no external frameworks)
3. ส่วนหน้าแรก (Home section) ให้ทำหน้าที่เป็น แกลเลอรีการ์ดสรุปหุ้น (Brief Portfolio Gallery) โดยการ์ดหุ้นแต่ละใบจะต้องแสดง:
   - ตัวย่อหุ้น Ticker โดดเด่นเป็นหัวข้อหลัก
   - วันที่บันทึกข้อมูลและวิเคราะห์รายงาน
   - ข้อความท่อนแรกความยาว 1-2 ประโยคจากหัวข้อประเมินธุรกิจ (Company snapshot) มาเป็นข้อความโปรยเรียกน้ำย่อย (Preview text)
4. เมื่อคลิกเลือกการ์ดหุ้นตัวใดตัวหนึ่ง ให้ระบบซูมแสดงผลหรือคลี่เนื้อหาบทวิเคราะห์ตัวเต็มของหุ้นตัวนั้นออกมาอ่านได้อย่างเป็นระเบียบ โดยการแปลงค่ารูปแบบ Markdown ให้แสดงผลออกมาเป็นหน้าหัวข้อ HTML ที่อ่านง่ายสบายตา (สามารถใช้วิธีจัดเก็บเนื้อหารายงานย่อยทั้งหมดฝังลงเป็นตัวแปรแบบจำลองภายในไฟล์ HTML เดียวกันเพื่อความเสถียรสูงสุดในการนำขึ้นระบบออนไลน์ฟรี)
5. สไตล์ CSS ตกแต่งให้เน้นความเรียบหรู ดูแพง ปลอดโปร่ง (Clean premium aesthetic) เช่น:
   - ใช้โทนสีแนวสว่างนุ่มนวลหรือดาร์กโหมดนิ่งหรู (harmonious curated palette)
   - ความกว้างเนื้อหาจำกัดเพื่อไม่ให้สายตาทำงานหนัก (max-width: 900px)
   - ระยะห่างช่องไฟขอบโค้งการ์ดและเงาดูโมเดิร์นทันสมัย (soft shadows, border-radius)
   - ใช้ฟอนต์สวยงามพรีเมียมจาก Google Fonts เช่น 'Inter' หรือ 'Outfit' เพื่อความหรูหรา น่าอ่าน

ทำเลยครับ เสร็จแล้วแจ้งความก้าวหน้าแผนงานให้ผมทราบด้วยนะครับ
````

## Step 4: อัปโหลดโฟลเดอร์แกลเลอรีของคุณขึ้นอินเทอร์เน็ต

## Step 6: ถอดรหัสทำความเข้าใจ "AgyOS" สถาปัตยกรรม AI ส่วนตัว

ยินดีด้วยอย่างยิ่งครับ! หน้าเว็บพอร์ตโฟลิโอบทวิเคราะห์ออนไลน์พรีเมียมของคุณเสร็จสิ้นสมบูรณ์แล้ว คราวนี้เรามาลองซูมภาพออกมาดูสิ่งที่พวกเราทำร่วมกันมาตั้งแต่ Lesson 1-4 ครับ จริง ๆ แล้วของทุกชิ้นที่คุณสร้าง ไม่ว่าจะเป็นปุ่มลัด, สกิล, พนักงานย่อย, สมองข้อมูล หรือแกลเลอรีเว็บ ทั้งหมดมันคือ **"ระบบปฏิบัติการ AI ขนาดเล็กของตัวเอง (AI-powered Operating System)"** ที่ซ้อนประสานกันเป็นชั้นอย่างเป็นระเบียบ ซึ่งผมขอตั้งชื่อเรียกอย่างเก๋ไก๋ว่า **AgyOS** ครับ!

สถาปัตยกรรมชั้นเชิงระบบของ **AgyOS** ที่สะสมขึ้นมาเป็นชั้น ๆ:

```text
┌─────────────────────────────────────────────────────────────┐
│  5. Public Showcase Web  →  Vercel Online (Lesson 5)        │  ← หน้าเว็บแบ่งปันให้ชาวโลก
├─────────────────────────────────────────────────────────────┤
│  4. Data Sources Memory  →  Sources/ 10-K & Transcripts     │  ← คลังข้อมูลข้อเท็จจริงจริง
│                             (Lessons 3-4)                   │    ที่ AI อ่านก่อนตอบ
├─────────────────────────────────────────────────────────────┤
│  3. Professional Services → Sub-agents Team (Lesson 4)       │  ← พนักงานย่อยแยกสายคิด
├─────────────────────────────────────────────────────────────┤
│  2. Operating Directives  → AGY.md Voice + SKILL.md SOP      │  ← กติกาควบคุมทิศทาง
│                             (Lessons 1-2)                   │    น้ำเสียงและสไตล์
├─────────────────────────────────────────────────────────────┤
│  1. Hardware & Kernel    → Antigravity (agy) Shell          │  ← แกนระบบหลักคอยสั่งรัน
└─────────────────────────────────────────────────────────────┘
```

การแยกและจัดระบบเป็นชั้น (Layers) แบบนี้มีความสำคัญสูงสุดในการขยายผลระบบในอนาคตครับ:
* หากวันหน้าคุณต้องการให้รายงานหุ้นเขียนวิเคราะห์ได้ฉลาดและกระชับขึ้น คุณเข้ามาแก้เฉพาะสีกติกาในชั้น **Operating Directives (สกิล SKILL.md)**
* หากวันหน้าคุณอยากอัปเดตตัวเลขให้ทันตามสถานการณ์ใหม่ คุณเพียงเข้าไปจัดการใส่ไฟล์ป้อนข้อมูลเพิ่มในชั้น **Data Sources Memory (โฟลเดอร์ sources/)** โดยไม่ต้องไปยุ่งเกี่ยวกับตัวโปรแกรม
* หากวันหน้าคุณอยากเพิ่มการวิเคราะห์เชิงเทคนิคและกราฟราคาหุ้น คุณเพียงสร้างไฟล์พนักงานสายกราฟเพิ่มขึ้นในชั้น **Professional Services (โฟลเดอร์ agents/)** เท่านั้น!

## Step 7: 3 ตัวอย่างไอเดียการนำรูปแบบ AgyOS ไปประยุกต์สร้างงานรูปแบบอื่น

กระบวนการออกแบบ "ปุ่มลัด + สกิล SOP + แยกพนักงานย่อย + ป้อนแฟ้มข้อมูลจริง + แสดงผลหน้าเว็บ" ที่คุณเพิ่งพัฒนาเสร็จในห้องเรียนนี้ ไม่ได้จำกัดอยู่แค่งานวิเคราะห์หุ้นเท่านั้นครับ แต่มันคือโครงพิมพ์เขียวเอนกประสงค์ (Universal Blueprint) ที่คุณสามารถนำไปติดตั้งกับอะไรก็ได้ในที่ทำงานของคุณ:

### ไอเดียที่ 1: ระบบลาดตระเวนประวัติตลาด (Watchlist Scout System)
แทนที่คุณจะส่งคำสั่งวิเคราะห์หุ้นทีละตัว คุณสามารถดัดแปลง สกิล ให้รับก้อนรายชื่อหุ้นที่คุณเฝ้าจับตามองอยู่ 10 ตัว สั่งพนักงานย่อยวิเคราะห์งบและข่าวพร้อมกัน แล้วคัดเฉพาะหุ้นที่มีสัญญาณการเปลี่ยนแปลงเชิงลึกเด่นที่สุดในวัน มาบันทึกสรุปเป็นหน้ารายการเตือนภัยส่วนตัว (Daily Scout Digest)

### ไอเดียที่ 2: ระบบคัดกรองสรุปข้อมูลผู้ให้บริการและคู่ค้า (Vendor/Vendor Brief Builder)
หากคุณทำงานจัดซื้อหรือต้องสแกนคัดกรองคู่ค้าทางธุรกิจ คุณสามารถสร้างพื้นที่ป้อนแฟ้มข้อมูลคุณสมบัติผลงานของแต่ละเจ้า ( sources/ ) สั่งพนักงานสแกนจุดแข็ง, จุดเสี่ยงทางการเงิน, และประเมินสัญญาจ้าง พร้อมออกรายงานเปรียบเทียบในโฟลเดอร์สรุป เพื่อเปิดประเมินความคุ้มค่าก่อนเซ็นสัญญาได้อย่างมีเหตุมีผลสูงสุด

### ไอเดียที่ 3: ระบบสแกนพิจารณาผู้สมัครงานและใบสมัคร (Hiring and Candidate Evaluator)
สำหรับงานฝ่ายบุคคล คุณสามารถสร้างท่อรับเรซูเม่และโปรไฟล์ผลงานของผู้สมัคร สั่งตั้งค่า สกิล ดัดแปลงให้อ่านเกณฑ์ทักษะที่แผนกต้องการ ประเมินความเชี่ยวชาญ ค้นหาผลงานที่เป็นประจักษ์ แล้วรายงานจุดน่าประทับใจและ 3 คำถามสัมภาษณ์คม ๆ ที่ควรใช้ยิงทดสอบผู้สมัครในห้องคุยจริง

## Step 8: เช็คลิสต์ตรวจสอบผลงานจบคอร์สอย่างสมบูรณ์

ยินดีด้วยอย่างยิ่งครับ! คอร์สเรียนหลักวิชาสร้างระบบปฏิบัติการ AI ส่วนตัวบน Antigravity (agy) ของเราดำเนินมาถึงฉากจบสุดท้ายอย่างยิ่งใหญ่แล้วครับ ก่อนจะปิดประตูกล่องหน้าต่างที่ 2 เรามาทบทวนสิ่งที่คุณทำสำเร็จมาทั้งหมด 5 บทเรียนกันครับ:

1. **Lesson 1**: เปิดพื้นที่ทำงานแรก สร้างไฟล์ควบคุมทิศทาง `AGY.md` และติดตั้งปุ่มลัด `/brief` v0 ตัวแรก
2. **Lesson 2**: ยกฐานะปุ่มลัดขึ้นเป็นระบบ สกิล SOP และใส่จิตวิญญาณสไตล์การเลือกหุ้นของคุณจริงลงใน `AGY.md`
3. **Lesson 3**: พัฒนาความซื่อสัตย์ของ AI โดยป้อนแหล่งข้อมูลประชุมผลประกอบการจริงเข้าระบบแทนการปล่อยให้จินตนาการตัวเลขเอง พร้อมบริหารกระดาษทดสมอง Token
4. **Lesson 4**: แตกกิ่งท่อประมวลผลออกเป็นทีมงาน AI พนักงานย่อย 3 คน ทำงานประสานขนานกันวิเคราะห์เชิงลึก
5. **Lesson 5**: ประดิษฐ์สร้างอินเทอร์เฟซ แกลเลอรี และประมูลเอาขึ้นระบบคลาวด์ส่งแชร์ให้ทุกคนชมผลงานได้สำเร็จจริง!

ของทุกชิ้นและสิ่งที่คุณประดิดประดอยมาทั้งหมดตั้งแต่วันแรกพิสูจน์แล้วว่า **"คุณมีศักยภาพในการเป็นสถาปนิก AI ที่จัดระบบโครงสร้างงานให้เครื่องมือออโตเมชันวิเคราะห์เรื่องลึกซึ้งได้อย่างน่าประทับใจ"** เลิกบอกตัวเองได้แล้วนะครับว่าคุณทำเทคโนโลยีหรืองานระบบระดับนี้ไม่ได้!
